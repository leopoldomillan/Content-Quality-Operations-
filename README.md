# Content Quality Operations Dashboard

## Project Overview

This portfolio project analyses a simulated high-volume content review operation.

The objective is to evaluate content review accuracy, SLA performance, recurring errors, escalation volumes and the effect of calibration sessions on operational quality.

The dataset is entirely fictional and was created solely for learning and portfolio purposes. It does not contain confidential, proprietary or company data.

## Business Problem

A content operations team reviews a high volume of user-generated content every day.

Management requires better visibility into:

- Overall review accuracy
- SLA compliance
- Over-enforcement and under-enforcement
- Repeated quality errors
- Escalation volumes
- Average handling time
- Performance before and after calibration sessions
- Policy categories generating the greatest operational risk

## Project Objectives

The project aims to:

1. Create and clean a simulated operational dataset.
2. Define meaningful quality and SLA metrics.
3. Build an interactive Power BI dashboard.
4. Identify trends, anomalies and recurring error patterns.
5. Conduct root-cause analysis.
6. Recommend practical process improvements.
